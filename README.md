# TaskFlow

## Описание проекта

TaskFlow - это приложение для управления задачами, предназначенное для командной работы. Приложение позволяет создавать задачи с использованием шаблонов, совместно работать с общим списком задач и отслеживать ход выполнения каждой задачи. Программа реализована с использованием архитектуры MVC, что обеспечивает четкое разделение логики, данных и интерфейса. Хранение данных осуществляется в структурированном текстовом файле. В основе приложения лежат три паттерна проектирования: Prototype (для клонирования шаблонов задач), Iterator (для навигации по списку задач) и State (для управления состояниями задач).

## Постановка задачи

Необходимо описать постановку задачи, решающую прикладную проблему.

### Проблема

В современной командной работе часто возникает необходимость быстро создавать типовые задачи, удобно перемещаться по списку задач и четко отслеживать статус выполнения каждой задачи. Существующие решения либо перегружены функционалом, либо требуют постоянного подключения к интернету, либо не позволяют гибко настраивать шаблоны задач.

### Цель разработки

Разработать программное обеспечение для управления задачами с тремя функциями - создание задач, совместная работа команды и отслеживание выполнения. Программа должна использовать архитектуру MVC и применять паттерн Prototype для клонирования шаблонов задач, паттерн Iterator для перехода от одной задачи к другой и паттерн State для управления различными состояниями задач. В качестве базы данных для хранения данных о задачах должен использоваться структурированный текстовый файл.

### Функциональные требования

1. **Создание задач с использованием Prototype** - пользователь должен иметь возможность создать шаблон задачи (название, описание, дата выполнения, приоритет) и клонировать его для быстрого создания новых задач без повторного ввода данных.

2. **Совместная работа команды с использованием Iterator** - пользователь должен иметь возможность последовательно переходить от одной задачи к другой в общем списке с помощью навигации "следующая задача" / "предыдущая задача".

3. **Отслеживание выполнения с использованием State** - пользователь должен иметь возможность отслеживать ход выполнения каждой задачи и изменять её состояние (новая, в работе, на проверке, завершена) с автоматическим ограничением доступных действий в зависимости от текущего состояния.


### Результат работы

Программа должна представлять собой консольное или оконное приложение, позволяющее пользователю:
- создавать шаблоны задач и клонировать их
- просматривать список задач с возможностью последовательного перехода между ними
- изменять состояние каждой задачи
- сохранять и загружать данные из текстового файла
